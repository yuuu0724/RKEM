# 集成检测系统工程实现计划

## Context

用户需要创建一个新的集成工程，结合芯片OCR和疲劳检测两个项目的功能，并预留芯片缺陷检测功能。该工程需要：
- 采用进程+线程架构，模块化设计
- 使用进程间通信(IPC)和线程间通信
- OCR模型运行在NPU Core 0，疲劳检测在Core 1，缺陷检测在Core 2
- 预留串口通信接口
- 三个独立的log文件记录识别结果
- 采用fatigue-demo的目录结构
- 详细注释和debug日志

## 系统架构设计

### 进程架构
```
┌─────────────────────────────────────────────────────────────┐
│                     主进程 (Main Process)                      │
│  - 管理所有子进程生命周期                                        │
│  - 处理用户输入和退出信号                                        │
│  - 监控子进程状态                                               │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ OCR子进程     │    │ 疲劳检测子进程 │    │ 缺陷检测子进程 │
│ (Process 1)  │    │ (Process 2)  │    │ (Process 3)  │
│ NPU Core 0   │    │ NPU Core 1   │    │ NPU Core 2   │
└──────────────┘    └──────────────┘    └──────────────┘
        │                     │                     │
        └─────────────────────┼─────────────────────┘
                              │
                    ┌─────────┴─────────┐
                    │   IPC通信层        │
                    │ (Unix Socket/管道) │
                    └───────────────────┘
```

### 线程架构（每个子进程内部）
```
┌─────────────────────────────────────────────────────────────┐
│                    子进程 (Sub Process)                       │
│                                                             │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐   │
│  │ 摄像头采集线程 │───▶│ 推理线程池    │───▶│ 结果处理线程  │   │
│  │ (Thread 1)   │    │ (Thread Pool)│    │ (Thread 2)   │   │
│  └──────────────┘    └──────────────┘    └──────────────┘   │
│          │                   │                   │         │
│          ▼                   ▼                   ▼         │
│  ┌──────────────────────────────────────────────────────┐  │
│  │              共享数据区 (Shared Data)                  │  │
│  │  - 帧缓冲区 (Frame Buffer)                           │  │
│  │  - 推理结果 (Inference Results)                       │  │
│  │  - 日志队列 (Log Queue)                               │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  ┌──────────────┐    ┌──────────────┐                      │
│  │ 日志写入线程   │    │ IPC通信线程   │                      │
│  │ (Thread 3)   │    │ (Thread 4)   │                      │
│  └──────────────┘    └──────────────┘                      │
└─────────────────────────────────────────────────────────────┘
```

### 进程间通信(IPC)机制
- **Unix Domain Socket**: 用于进程间命令和结果传递
- **共享内存**: 用于高效图像帧共享
- **信号量**: 用于进程同步

### 线程间通信机制
- **互斥锁 (Mutex)**: 保护共享数据
- **条件变量 (Condition Variable)**: 线程通知
- **无锁队列 (Lock-free Queue)**: 高效数据传递
- **线程池**: 管理多个推理任务

## 日志系统设计

### 日志文件
1. **ocr_results.log**: 记录OCR识别结果
   - 格式: `[时间戳] 识别文本: xxx, 置信度: 0.xx, 边界框: [x1,y1,x2,y2]`
   
2. **fatigue_results.log**: 记录疲劳检测结果
   - 格式: `[时间戳] 检测类别: xxx, 置信度: 0.xx, 状态: 正常/疲劳`
   
3. **defect_results.log**: 记录缺陷检测结果（预留）
   - 格式: `[时间戳] 缺陷类型: xxx, 置信度: 0.xx, 位置: [x,y,w,h]`
   
4. **debug.log**: 记录调试信息
   - 包含模型加载、推理时间、错误信息等

### 日志级别
- **LOG_DEBUG**: 调试信息
- **LOG_INFO**: 普通信息
- **LOG_WARN**: 警告信息
- **LOG_ERROR**: 错误信息

## 串口通信接口设计

```cpp
// 串口配置
struct SerialConfig {
    std::string port;           // 串口设备路径，如 /dev/ttyS0
    int baudrate;               // 波特率，如 115200
    int databits;               // 数据位，如 8
    int stopbits;               // 停止位，如 1
    char parity;                // 校验位，如 'N'
};

// 串口通信接口
class SerialPort {
public:
    bool open(const SerialConfig& config);
    void close();
    int send(const uint8_t* data, size_t len);
    int receive(uint8_t* buffer, size_t len, int timeout_ms);
    bool isOpen() const;
};
```
